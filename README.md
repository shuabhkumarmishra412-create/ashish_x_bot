# 🚀 デプロイ ・ DEPLOYMENT 🚀

</div>

### 🚂 ・ DEPLOY ON RAILWAY

```bash
# 1️⃣  Clone or download this project
git clone <your-repo-url> && cd ashish_x_bot

# 2️⃣  Push to GitHub
git remote add origin <your-github-repo>
git push -u origin main

# 3️⃣  On Railway → New Project → Deploy from GitHub
#     ⚡ Bot starts within ~60 seconds. ⚡
```

### 🐳 ・ RUN LOCALLY

```bash
pip install -r requirements.txt
python main.py
```

# ⚡ ライティング ・ LIGHTNING NOTES ⚡

</div>

> ⚠️  **Token Safety** — `BOT_TOKEN` is hardcoded in `main.py`. For production, move it to a Railway environment variable.
>
> 💾  **Storage** — Railway's filesystem is ephemeral. JSON files reset on every redeploy. Use a Railway **Volume** or external database for permanent storage.
>
> 🔁  **Single Instance Only** — Telegram allows only one polling instance per token. Don't run two copies at once.
>
> 🧹  **Auto-Delete** — Every bot message self-destructs after **120 seconds**. Adjust `AUTO_DELETE_SECONDS` in `main.py` if you want a different timer.
